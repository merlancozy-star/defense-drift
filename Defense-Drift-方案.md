# Defense Drift：跨域投毒防御的可分性塌缩诊断

> **一句话定位**：现有 RAG 投毒防御都隐含假设"干净 / 投毒可分性"是域无关的常量；我们证明它其实是**域依赖**的——换到专业语料会系统性塌缩，导致这些防御失效——并给出可分性塌缩的度量与换域重测协议。

> **工作名**：Defense Drift（现象：防御阈值/可分性的跨域漂移）。方法侧协议暂称 DACR（Domain-Adaptive Calibration for RAG defenses）。名字可改。

---

## 0. 这份文档是什么 / 怎么用

这是一篇**会议论文（EI/IEEE 量级）的完整研究方案**，目标是在赶毕业的时间窗内（秋季投、年底到次年初录用）产出一篇"稳录、不撞车、与 Legal-TREND 不自我重复"的第二篇论文。

定位为 Legal-TREND（检索策略层）的**对抗可信层**分支：那篇解决"怎么检得动态、检得准"，这篇解决"检回来的东西在对抗污染下还可不可信"。两篇合起来拼成"可信 RAG"的完整研究身份。

用法：
- 作为 **proposal / 开题** 的底本；
- 作为 **CLAUDE.md** 的设计文档源（第 6 节的 YAML 可直接迁移）；
- 作为 **写作大纲**（第 3–5 节对应 Introduction / Related Work / Method，第 6 节对应 Experiments）。

## 1. 背景与动机

### 1.1 赛道现状：防御方法已经卷满

RAG 投毒攻防是 2024 下半年至今的高热度方向，防御侧已有一长串：TrustRAG（K-means 簇过滤 + 冲突消解）、RAGuard（困惑度 + 相似度过滤）、RobustRAG（isolate-then-aggregate 投票）、AstuteRAG（内外知识自适应融合）、SeCon-RAG（两阶段语义过滤）、SDAG（稀疏文档注意力）、RAGDefender（频率排序）、DRS / FilterRAG / Stealth Lens 的 AV Filter（基于可分性的检测）等，几乎每月一篇。

**关键事实**：这些防御被复现工作反复证明"残余风险仍在"——即便是 SOTA 也只能部分缓解，强自适应攻击（CorruptRAG、AuthChain、Confundo、DisarmRAG）仍能维持高攻击成功率。所以"再造一个更强的过滤器"这条路新意稀薄、且容易被新攻击打穿。

### 1.2 被忽视的变量：域

攻击侧已经证明**语料域是真实的安全变量**：Semantic Chameleon（2026.03）的案例研究显示，朴素攻击跨语料迁移成功率为 0%，而语料自适应攻击能拿到 100% 检索命中。专业语料（术语密集）会"吸收"攻击词汇，使基于词汇异常 / 困惑度的检测失灵。

但防御侧几乎所有工作的关键阈值与可分性假设，都是在通用域（NQ / HotpotQA）上调定的，**没有人系统验证过：这些防御换到专业域是否还成立**。Semantic Chameleon 自己也承认，以往工作虽跨多数据集评测，却没有显式量化语料相关的检测难度、没把语料域当成一等安全参数——并明确将其列为 future work。

**这就是本方案的空地：把"防御的跨域失效"从猜测变成可量化的诊断，并给出补救协议。**

## 2. 定位是怎么收敛到这里的（三轮 positioning 的教训）

本方案经过三轮文献核查后定型。记录于此，避免后续重蹈覆辙：

| 轮次 | 曾经的"方法新意" | 核查结果 |
|---|---|---|
| 一 | 风险自适应动态分档防御（RADAR） | 撞 ADO（Sentinel-Strategist, 2026.04），核心同构 |
| 二 | 分位数锚定阈值（DACR Step 2） | 早是 PPL 防御标配（99 分位、mean+2std） |
| 三 | 可分性驱动校准 + 探针 | DRS / FilterRAG / Stealth Lens 已做 |

**规律**：投毒防御的"造新方法"维度已被占满，每个信号都有主；而"防御换域失效 / 可分性的域依赖塌缩"这个**诊断**维度，三轮搜索始终无人正面做。

**结论**：本方案是**框架 / 诊断型论文**，不是"第 N 个防御方法"。方法仅作为服务于诊断的最小工具。写作时必须把读者注意力引向"问题与发现"的尺子，而非"方法新意"的尺子。

## 3. 核心主张（Introduction 的骨）

现有所有基于"干净 / 投毒可分性"的防御（DRS、FilterRAG、Stealth Lens 的 AV Filter、以及隐式依赖可分性的 PPL / TrustRAG / RAGuard）都隐含假设：**可分性是域无关的常量**。

我们证明它其实是**域依赖**的：

1. **诊断（主结果）**：在专业语料里，攻击词汇被领域词表吸收，干净与投毒在各防御所依赖的信号上的可分性显著塌缩；将通用域调好的防御直接搬到专业域，会出现可量化的双向退化（阈值太松→ASR 回升 / 太紧→clean ACC 下降）。
2. **根因**：现有可分性度量只锚定了"干净侧"分布，忽略了"投毒相对干净分布的位置"也随域变化。
3. **补救（方法）**：给出一个 defense-agnostic 的换域重测协议——换域时不仅重估干净分布，还要实测可分性；当可分性塌缩到不可用时，退到拒答 / 低置信兜底。

## 4. 与现有工作的切割（Related Work 的防守锁）

这是方案的安全锁，必须从设计阶段钉死，且写进论文对照表：

| 对照工作 | 它做了什么 | 本方案的差异 |
|---|---|---|
| **ADO**（Sentinel-Strategist, 2026.04） | query 级、多攻击向量：每条 query 选部署哪几种防御 | 我们是**域级**、**单攻击（投毒）**：不选"用哪种防御"，而是诊断"已有防御换域是否还成立"。维度正交——ADO 横向选防御，我们纵向跨域校准 |
| **Semantic Chameleon**（2026.03） | 攻击侧 + 观测：证明语料域影响攻防难度，把跨域防御列为 future work | 我们是**防御侧 + 方法**：正面接它的 future work，给出诊断度量与重测协议，不止于观测 |
| **Stealth Lens / AV Filter**（2510） | 把"可分性"形式化为单域内的可区分性安全博弈，证明有效投毒必然可分 | 我们把这个博弈**从单域推广到跨域**，证明可分性是域依赖的、换域会塌缩。是其框架的延伸，非另起炉灶 |
| **DRS / FilterRAG** | 基于干净/对抗的统计可分性做单域检测 | 我们不提新检测信号；把它们当作被诊断对象，测其可分性的跨域稳健性 |
| **来源可靠性估计**（2410.22954） | 把来源可信度建模进检索 | 我们不重做来源建模；只用检索分布统计量，更轻 |

**关键话术**：本方案是 *defense-agnostic 的诊断层*，让 TrustRAG / RAGuard / DRS 等从"被比下去的竞品"变成"被诊断与增强的对象"——Related Work 从对抗变互补，审稿阻力骤降。

**排雷**：DRS（Directional Relative Shifts）名字里也有 "shift"，但它是单域内沿"攻击易发方向"看偏移做检测，不是跨域阈值漂移。写作时主动区分，防止评审混淆。

## 5. 方法设计（Method，全部 training-free）

方法服务于诊断，最小化、不与任何信号竞争。

### 5.1 可分性度量（诊断的核心算子）

对每个防御所依赖的信号 `s`（困惑度 / DRS / 簇间距 / 注意力方差），在给定域 `d` 上：

1. 取该域一小批查询，跑检索得到干净 passage 集合；
2. 用攻击 harness 注入少量投毒探针，得到投毒 passage 集合；
3. 计算信号 `s` 在两个集合上取值分布的 **AUROC**（衡量"用 s 能多好地分开干净/投毒"）；
4. 定义 **可分性塌缩量**：`separability_drop(s) = AUROC_source(s) − AUROC_target(s)`。

`separability_drop` 是本论文的招牌量——它把"换域失效"变成一个可填进矩阵格子的数。

### 5.2 换域重测协议（DACR）

三个条件，构成必要性对照：

- **source_threshold**：直接搬源域阈值（下界，证明"不重测会失效"）。
- **clean_recalib**：仅在目标域重估干净分布的分位数阈值（强 baseline，审稿人会想到的做法）。
- **separability_recalib**：按目标域实测可分性调阈值；当 AUROC 低于可用阈（如 0.6）时，标记该域为"该信号不可靠"，触发拒答 / 低置信兜底。

**胜负手**：必须证明 `separability_recalib` 显著优于 `clean_recalib`——即"只重估干净侧不够，得看可分性"。这条对照是方法新意成立的唯一支点。

> 工程上整套零训练、零梯度，最重的计算是批量跑检索 + 算 AUROC + 扫阈值。正是 milrag 脚手架（多配置批量编排、YAML 控超参）的主场。

## 6. 实验设计（Experiments，直接映射 milrag config）

核心是两张表。

### 6.1 表一 · 可分性塌缩矩阵（主结果 / 招牌图）

```yaml
# separability_collapse.yaml — 核心诊断实验
signals:                  # 各防御赖以工作的可分性信号
  - perplexity            # PPL / RAGuard
  - drs                   # Directional Relative Shift
  - cluster_distance      # TrustRAG K-means 簇间距
  - attention_variance    # Stealth Lens AV Filter
domains:
  source:  [nq, hotpotqa]       # 信号原本调好的通用域
  target:  [legal, pubmedqa]    # 测塌缩的专业域
attack:
  - poisonedrag                 # 必做基准
  - corruptrag                  # 抗防御强攻击（可分性更低，放大塌缩）
metric:
  - auroc_clean_vs_poison       # 该信号在该域分干净/投毒的 AUROC
  - separability_drop           # = AUROC(source) − AUROC(target) ← 招牌量
poison_ratio: [0.05, 0.1, 0.2, 0.4]   # 顺带覆盖低/高投毒比例
```

格子填 `separability_drop`，一眼看出每个信号换到专业域都塌。**这张表本身就是论文主结果，不依赖任何新方法奏效。**

### 6.2 表二 · 重测必要性对照（证明协议价值）

```yaml
# recalibration_necessity.yaml
defenses: [ppl_filter, trustrag, raguard]
conditions:
  - source_threshold        # 直接搬源域阈值（下界）
  - clean_recalib           # 仅重估干净分布分位数（强 baseline）
  - separability_recalib    # 按目标域实测可分性调（我们的协议）
report: [asr, clean_acc, abstain_rate]
```

### 6.3 消融

- 去掉可分性项、退回纯 clean 分位数 → 证明可分性信息不可省；
- 探针数量敏感性 → 证明协议在极少探针下仍可用；
- 延迟测量 → 证明 training-free、开销可忽略。

### 6.4 指标口径

ASR（攻击成功率，↓）、clean ACC、受攻击 ACC、AUROC、separability_drop、abstain_rate、额外延迟。

## 7. 方案评估：效果 / 创新度 / 工作量

### 7.1 效果预测

- **最可能结局（~70%）**：结构干净、实证扎实的会议论文，borderline-accept 到 accept。对毕业、简历足够。
- **三重保险**（任意一重成立都不至全盘皆输）：
  1. 诊断结果本身即结论——塌缩矩阵显著就可发表，不依赖方法奏效；
  2. 反向结果也是好结果——若某信号换域不塌，则得到"哪类信号域稳健"的判据；
  3. 站在 Stealth Lens 最新框架上做跨域推广，天然显得 informed。
- **唯一真实风险**：塌缩幅度可能不够"惊人"（如 AUROC 仅 0.85→0.80）。
  - 缓解：攻击侧用更隐蔽的 CorruptRAG；专业语料选术语越密集越好（法律/医疗）；第 3 周单条迁移验证即可提前看苗头。

### 7.2 创新度（诚实打分）

| 维度 | 评级 | 说明 |
|---|---|---|
| 问题新颖性 | **高** | "可分性域依赖塌缩"三轮无人正面做，有 future work 背书 |
| 方法新颖性 | **低偏中** | 各步骤有前人；价值在"首个跨域诊断框架"的组织，而非新算子 |
| 实证价值 | **高** | 跨信号 × 跨域 × 跨投毒比例的塌缩矩阵是社区缺的资产 |

定位：**"重新框定被忽视问题"型创新**，而非"更强方法"型。会议受欢迎，难进顶会最高分段。对当前目标（稳录、毕业）是对的；若日后冲 ACL/EMNLP 主会，需在方法侧再加重拳。

### 7.3 工作量（8 周可扛，比 RADAR 更省）

| 模块 | 占比 | 说明 |
|---|---|---|
| 复现现有防御 baseline | ~35% | 最重；需复现各自源域数字。多为 training-free + 开源，是"接管线"非"从零写" |
| 注入与评测管线 | ~20% | PoisonedRAG harness + AUROC + 跨域跑批，一次写好反复用 |
| 两张表批量实验 | ~30% | 笛卡尔积配置量大但全并行，YAML config 主场 |
| 写作 + 消融 | ~15% | 会议体量，留一周写作 |

**工程暗礁**：专业域语料获取与清洗。须保证目标域的检索管线 / embedding / chunk 策略与源域一致，否则"塌缩"被语料工程差异污染成 confound。预留 2–3 天缓冲。

## 10. 一页速查

- **是什么**：跨域投毒防御的"可分性塌缩"诊断框架 + 换域重测协议。
- **新在哪**：问题新（可分性是域依赖的、换域塌缩，无人正面做）；方法不新（刻意，避免撞车）。
- **怎么证**：表一塌缩矩阵（主结果）+ 表二重测必要性对照（胜负手：separability_recalib > clean_recalib）。
- **为什么稳**：诊断结果本身即结论 + 反向结果也是好结果 + 站在 Stealth Lens 框架上。
- **多大量**：8 周，training-free，milrag 脚手架射程内；前两周最重，建议入职前啃完。
- **值多少**：稳录会议 + 干净延续"可信 RAG"研究身份；不冲顶会。
- **下一步**：落实表一 `auroc_clean_vs_poison` 的算法骨架（信号打分函数 / 投毒注入 / AUROC 聚合）。
